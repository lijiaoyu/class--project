## 1.2 背景

当前国内高校校园文化建设持续推进，校园文创产品作为文化传播的重要载体，市场需求逐年增长。但多数高校的文创产品仍依赖线下实体店、校园集市等传统渠道销售，存在展示形式单一、用户交互不足、传播范围有限、数据化管理缺失等问题，无法满足师生及校友对校园文化产品的多元化体验需求。

本项目旨在开发一套校园文创产品展示与交互系统，通过线上平台实现文创产品的3D展示、互动体验、在线购买、用户反馈等功能，打通校园文创产品的线上传播与销售链路，助力校园文化传播，提升文创产品的影响力与商业价值。

## 1.3 项目概述

本系统是面向高校师生、校友及社会游客的校园文创产品线上平台，核心功能包括：文创产品分类展示、3D/AR交互预览、用户评价与互动、订单管理、文创资讯推送、后台商品与用户管理等。系统采用前后端分离架构，前端支持Web端与移动端适配，后端提供数据管理与业务逻辑支持，目标是打造集展示、交互、交易、传播于一体的校园文创服务平台。

## 1.4 文档概述

本报告为校园文创产品展示与交互系统的可行性分析报告，旨在对项目的技术、经济、法律、用户使用等维度进行全面评估，明确项目开发的必要性与可行性，为后续系统设计与开发提供决策依据。报告内容涵盖项目背景、需求分析、可选方案对比、各维度可行性评估及最终方案选择。

# **3** 可行性分析前提

## 3.1 项目要求

1. 功能需求：实现文创产品的多形式展示（图文、3D、AR）、用户交互（评论、收藏、分享）、在线交易（下单、支付、订单追踪）、后台管理（商品、用户、订单、数据统计）四大核心模块。

2. 性能需求：系统支持500人同时在线访问，页面响应时间≤2秒，3D模型加载时间≤5秒，移动端适配主流手机型号。

3. 安全需求：用户信息加密存储，支付流程符合第三方支付安全规范，防止数据泄露与非法访问。

4. 兼容性需求：Web端支持Chrome、Edge、Firefox等主流浏览器，移动端适配iOS 12+、Android 8.0+系统。

## 3.2 目标

1. 短期目标：完成系统核心功能开发，实现校园文创产品的线上展示与基础交互，在本校范围内上线测试，覆盖500+师生用户。

2. 中期目标：优化系统性能与用户体验，完善AR交互、用户社区等功能，接入校园官方文创渠道，实现年销售额突破10万元。

3. 长期目标：打造具有本校特色的校园文创IP平台，逐步向周边高校推广，成为区域校园文创传播标杆平台。

## 3.3 环境、条件、假定和限制

1. 硬件环境：开发端为普通PC（i5处理器、8GB内存、512GB硬盘），服务器采用阿里云轻量应用服务器（2核4GB内存、100GB云盘），满足系统开发与初期部署需求。

2. 软件环境：前端采用Vue.js + Three.js/AR.js框架，后端采用Spring Boot框架，数据库使用MySQL，开发工具为VS Code、IDEA，均为开源免费工具，无版权成本。

3. 人员条件：开发小组由3-4名软件工程专业学生组成，具备基础的前后端开发能力，指导老师提供技术支持。

4. 假定与限制：

- 假定学校文创管理部门能提供产品素材、授权信息及合作支持；

- 假定第三方支付接口（如支付宝校园支付）可顺利接入；

- 限制：项目开发周期为3个月，开发预算控制在5000元以内（主要为服务器与域名费用）；

- 限制：初期仅支持本校IP访问，暂不对外开放社会用户注册。

## 3.4 可行性分析方法

本报告采用结构化分析方法，结合调研法、对比分析法、成本效益分析法，从技术、经济、法律、用户使用四个维度对项目进行评估，通过梳理需求、列举可选方案、对比各方案优劣，最终确定最优开发方案。

# **4** 可选的方案

## 4.1 原有方案的优缺点、局限性及存在的问题

目前本校校园文创产品的主要运营方案为：线下实体店+校园集市+微信公众号图文推送。

- 优点：用户可现场体验产品，信任度高；运营成本低，无需复杂技术开发。

- 局限性：展示形式单一，仅为平面图文或实物陈列，无法体现产品细节；传播范围仅限校内，校友及校外用户难以参与；缺乏用户互动渠道，无法收集用户反馈；数据化管理缺失，难以统计产品销售与用户偏好数据。

- 存在的问题：受时间、空间限制明显，校园集市仅在特定时间举办；线下实体店运营成本高，销量受客流影响大；无法满足年轻用户对线上交互、沉浸式体验的需求。

## 4.2 可重用的系统，与要求之间的差距

目前市面上的通用电商平台（如淘宝、微店）可实现基础的商品展示与交易功能，但与本项目需求存在明显差距：

- 缺乏校园特色交互功能，无法实现3D/AR文创产品预览；

- 无法对接校园用户体系，无法实现校内专属优惠、校友认证等功能；

- 通用平台的社区功能与校园文创场景适配性差，难以打造专属用户社区；

- 数据统计功能无法针对校园文创场景定制，无法满足学校对文创产品运营数据的管理需求。

## 4.3 可选择的系统方案1：轻量版Web平台+微信小程序

1. 架构设计：前端开发轻量Web页面与微信小程序，后端采用Spring Boot + MySQL架构，部署在阿里云轻量服务器上。

2. 核心功能：实现文创产品图文展示、用户收藏/分享、在线下单、基础后台管理功能，暂不开发3D/AR交互模块。

3. 技术特点：开发周期短（1.5个月），技术难度低，开发成本低，适配移动端用户习惯。

4. 优缺点：

- 优点：快速上线，满足基础展示与交易需求，用户门槛低，微信小程序无需下载即可使用。

- 缺点：缺乏沉浸式交互体验，无法体现校园文创产品的设计细节，与通用电商平台差异化不足。

## 4.4 可选择的系统方案2：完整版Web平台+移动端适配（含3D/AR交互）

1. 架构设计：采用前后端分离架构，前端Web端使用Vue.js开发，集成Three.js实现3D产品展示、AR.js实现AR交互功能，移动端适配响应式页面；后端采用Spring Boot + Redis架构，实现高并发访问支持，数据库使用MySQL + Redis缓存。

2. 核心功能：实现文创产品3D/AR交互预览、用户评价与社区互动、在线交易、订单管理、后台数据统计、校园用户认证等完整功能。

3. 技术特点：功能完整，用户体验丰富，具备校园文创平台的差异化优势，可扩展性强，后续可接入更多校园文创场景。

4. 优缺点：

- 优点：用户体验好，沉浸式交互提升产品吸引力，数据化管理完善，便于后续运营优化，平台差异化明显。

- 缺点：开发周期较长（3个月），技术难度较高，对开发人员的前端交互开发能力有一定要求，服务器成本略高于方案1。

## 4.5 选择最终方案的准则

1. 功能完整性：是否满足项目核心需求，具备校园文创平台的特色功能；

2. 技术可行性：开发团队是否具备实现方案的技术能力，技术选型是否成熟稳定；

3. 成本效益：开发与运营成本是否在预算范围内，是否能实现预期收益；

4. 用户体验：是否适配目标用户的使用习惯，能否提供差异化的交互体验；

5. 可扩展性：是否便于后续功能迭代与平台推广。

综合以上准则，选择方案2（完整版Web平台+移动端适配）作为最终开发方案，在控制开发成本的前提下，优先实现核心的3D/AR交互与基础交易功能，保障平台的差异化与用户体验。

# **5** 所建议的系统

## 5.1 对所建议的系统的说明

校园文创产品展示与交互系统是一套面向高校校园文创场景的线上服务平台，采用前后端分离架构，分为用户端与后台管理端两部分：

- 用户端：支持Web端与移动端访问，提供文创产品展示、3D/AR交互预览、用户注册/登录、收藏/分享、评价互动、在线下单、订单追踪、文创资讯浏览等功能。

- 后台管理端：面向文创管理部门与运营人员，提供商品管理、用户管理、订单管理、数据统计、内容审核等功能，支持多角色权限分配。

系统整体采用模块化设计，便于后续功能扩展与维护，同时接入校园统一身份认证系统，实现校内用户一键登录，提升用户使用便捷性。

## 5.2 数据流程和处理流程

### 1. 用户端数据流程：

用户注册/登录 → 浏览文创产品 → 查看3D/AR交互预览 → 收藏/评价/分享产品 → 下单支付 → 生成订单 → 订单追踪 → 收货确认 → 用户反馈

### 2. 后台数据流程：

运营人员登录后台 → 上传/更新文创产品信息（含3D模型、图文素材） → 审核用户评价 → 处理订单信息 → 统计销售与用户数据 → 发布文创资讯 → 管理用户权限

### 3. 核心数据处理流程：

用户提交订单后，系统自动校验库存与用户信息，调用第三方支付接口完成支付，支付成功后生成订单并同步至后台，同时更新库存数据；用户评价提交后，需经后台审核通过后才会在前端展示，确保内容合规。

## 5.3 与原系统比较

对比维度 原系统（线下+微信公众号） 本项目系统

运营成本 线下实体店、集市成本高 线上平台运营成本低，可长期使用

## 5.4 影响（或要求）

1. 对用户的影响：为师生、校友提供便捷的校园文创产品获取渠道，沉浸式交互体验提升用户参与感，校园文化传播形式更丰富多元。

2. 对运营方的影响：实现文创产品的线上数字化管理，降低线下运营成本，通过数据统计优化产品设计与运营策略，提升文创产品的销量与影响力。

3. 对开发环境的影响：需搭建前后端开发环境，配置服务器与数据库，接入第三方支付、AR交互等相关接口。

4. 对学校管理的影响：需与学校文创管理部门、信息中心沟通协调，获取产品素材、校园用户认证授权等支持。

## 1. 版权合规性：

​

- 系统开发使用的开源框架与工具均符合开源协议要求，无版权侵权风险；

​

- 文创产品素材、图片、3D模型均由学校文创管理部门提供，拥有合法版权，系统仅用于校园内非商业性运营，不侵犯第三方知识产权。

​

## 2. 数据合规性：

​

- 严格遵守《个人信息保护法》，用户信息采用加密存储，仅收集必要的用户数据，不泄露用户隐私；

​

- 接入校园统一身份认证系统，用户登录信息由学校统一管理，符合校园数据安全管理规范。

​

## 3. 运营合规性：

​

- 系统在线交易功能仅用于校园文创产品销售，不涉及非法交易；

​

- 后台设置内容审核机制，防止违规信息发布，符合校园网络安全管理要求。

综上，本项目系统在版权、数据、运营等方面均符合相关法律法规与学校管理规定，法律可行性较高。

- 线下校园集市、公众号等渠道可同步推广系统，引导用户注册使用，提升用户接受度。
