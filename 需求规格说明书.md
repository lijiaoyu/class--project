3. 项目目标与内容

3.1 项目目标

构建一个校园文创展示与轻交互平台，提升校园文化氛围，促进文创产品的可见度与定制转化率。

3.2 核心内容

模块 内容说明

展示模块 产品图文展示、分类筛选、关键词搜索

交互模块 点赞、评论、收藏

定制模块 定制需求提交、创作者回复

个人空间 我的收藏、我的定制、我的作品

后台管理 产品审核、用户管理、评论审核

---

4. 功能需求

4.1 展示模块

· 首页展示热门推荐文创产品（轮播图 + 推荐列表）

· 按类别筛选：服饰、文具、数码周边、纪念品

· 支持关键词模糊搜索（产品名称、创作者）

4.2 交互模块

· 用户可对产品进行点赞与取消点赞

· 用户可发表评论，支持删除自己的评论

· 用户可收藏产品至个人空间

4.3 定制模块

· 用户填写定制需求单：产品名称、需求描述、参考图片（可选）

· 创作者查看定制申请并回复

· 用户查看定制状态：待回复 / 已回复

4.4 个人空间页面

· 展示个人信息：头像、昵称、学号

· 我的收藏列表：展示已收藏产品，支持取消收藏

· 我的定制记录：展示定制历史及回复状态

· 我的作品（仅创作者可见）：管理自己上传的产品

4.5 后台管理

· 产品审核：审核通过后上架展示，不通过则退回并说明原因

· 用户管理：禁用/恢复用户账号

· 评论审核：敏感词自动过滤，人工审核异常评论

---

5. 非功能需求

类别 具体要求

安全性 学号+密码认证登录；上传图片校验格式（jpg/png）与大小（≤5MB）；评论敏感词过滤；用户密码加密存储

可靠性 系统可用性 ≥ 99%；数据每周全量备份，每日增量备份；支持故障恢复

灵活性 支持后续扩展为小程序或移动端H5；产品分类标签可后台动态配置；支持新增产品类型

性能 首页加载时间 ≤ 2秒；支持同时在线用户 ≥ 200人；搜索响应时间 ≤ 1秒

易用性 界面简洁直观；操作流程不超过3步；支持移动端响应式布局

6. 页面设计说明

页面名称 核心内容 主要功能点

首页 顶部轮播图、推荐文创区、分类入口 快速浏览热门产品、跳转分类

产品详情页 产品多图展示、基本信息、点赞评论区 查看详情、点赞评论、收藏、发起定制

登录/注册页 学号+密码登录、注册表单 用户身份认证

7. 其他需求整合

需求类型 具体说明

架构要求 采用前后端分离架构（如 Vue + Spring Boot），便于迭代与维护

数据库 遵循第三范式设计，保证数据一致性

接口文档 提供 RESTful API 接口说明文档

移动适配 响应式布局，支持手机、平板、PC 访问

部署环境 支持 Linux 服务器部署，使用 MySQL + Redis

开发规范 遵循 MVC 设计模式，代码注释覆盖率 ≥ 30%
