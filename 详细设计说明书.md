# **4** 体系结构设计

## 4.1 体系结构

采用分层架构，分为表现层、业务逻辑层、数据访问层：

- 表现层：用户界面、管理员后台界面

- 业务逻辑层：用户管理、商品管理、订单管理、评论管理业务逻辑

- 数据访问层：数据库增删改查操作封装

### 4.1.1 程序模块划分

- 用户模块：注册、登录、个人信息管理

- 商品模块：商品展示、3D预览、商品搜索

- 订单模块：下单、订单查询、订单管理

- 评论模块：发表评论、评论审核

- 管理员模块：商品管理、订单处理、用户管理

### 4.1.2 模块层次结构关系

表现层调用业务逻辑层接口，业务逻辑层调用数据访问层接口，实现数据交互。

## 4.2 全局数据结构说明

- 用户信息：存储用户账号、密码、个人信息

- 商品信息：存储商品ID、名称、价格、库存、图片、3D模型地址

- 订单信息：存储订单号、用户ID、商品ID、下单时间、订单状态

- 评论信息：存储评论ID、用户ID、商品ID、评论内容、评分、审核状态

## 4.3 部件设计（核心模块详细设计）

### 模块1：用户登录模块

- 功能描述：验证用户账号密码，实现用户登录；未注册用户可跳转注册页面。

- 输入：用户名、密码

- 输出：登录成功跳转首页；登录失败提示“用户名或密码错误”

- 程序逻辑：

1. 用户输入账号密码，点击登录；

2. 系统校验输入格式，调用数据库查询用户信息；

3. 比对密码，验证通过则生成会话标识，跳转首页；验证失败返回错误提示。

- PlantUML流程图代码：

### 模块2：商品下单模块

- 功能描述：用户选择商品，提交订单，完成下单流程。

- 输入：用户ID、商品ID、购买数量、收货地址

- 输出：订单生成成功，返回订单号；库存不足提示“商品库存不足”

- 程序逻辑：

1. 用户选择商品，确认购买数量；

2. 系统查询商品库存，校验库存数量；

3. 库存充足则生成订单，扣减商品库存；库存不足返回错误提示；

4. 订单生成成功，返回订单信息给用户。

- PlantUML流程图代码：

### 模块3：商品上架模块（管理员）

- 功能描述：管理员录入商品信息，提交审核，上架商品。

- 输入：商品名称、价格、库存、图片、3D模型地址、分类

- 输出：商品信息保存成功，进入审核状态；审核通过后商品上架展示。

- 程序逻辑：

1. 管理员录入商品信息，提交系统；

2. 系统校验信息完整性，保存商品数据，状态设为“待审核”；

3. 管理员审核通过后，商品状态更新为“已上架”，前端可展示。

## 5.4 数据库表结构设计

username varchar(20) 非空 用户名

password varchar(32) 非空 登录密码

phone varchar(11) 唯一 联系电话

### (2) 商品表（product）

表格

字段名 数据类型 约束 说明

price decimal(10,2) 非空 商品价格

stock int 非空 商品库存

p\_status varchar(10) - 商品状态（待审核/已上架/已下架）

p\_img varchar(100) - 商品图片路径

user\_id int 外键 用户ID

product\_id int 外键 商品ID

order\_time datetime - 下单时间

order\_status varchar(10) - 订单状态
